# JavaScript SDK (`@paynodelabs/sdk-js`)

The JavaScript SDK provides both **client-side** tools for AI Agents (Payer) and **server-side** middleware for Merchants (Receiver).

## 📦 Installation

```bash
npm install @paynodelabs/sdk-js ethers
# Optional: for production Redis idempotency
npm install ioredis
```

---

## 🤖 For AI Agents (Client)

> **⚠️ Prerequisites**
>
> The Agent's wallet must hold **Base ETH** (for gas) and **USDC** (for payment).
> First time? See the [Contracts](/contracts) page for testnet faucet instructions.

The `PayNodeAgentClient` automatically handles `402` responses, executes USDC payments on-chain, and retries the original request.

```typescript
import { PayNodeAgentClient } from '@paynodelabs/sdk-js';

// Initialize with private key and RPC
const agent = new PayNodeAgentClient(
  process.env.AGENT_PRIVATE_KEY,
  'https://mainnet.base.org'
);

// Make requests — 402 payment is handled autonomously
const response = await agent.requestGate('https://api.merchant.com/premium-data');
const data = await response.json();
console.log('✅ Data:', data);
```

### EIP-2612 Permit (Single-Tx Payment)

For gasless (from the payer's perspective) single-transaction payments using EIP-2612 Permit:

```typescript
const agent = new PayNodeAgentClient(PAYER_PRIVATE_KEY, RPC_URL);

// 1. Payer signs a permit offline (no gas needed)
const { deadline, v, r, s } = await agent.signPermit(
  USDC_ADDRESS,                    // Token
  PAYNODE_ROUTER_ADDRESS,          // Spender (Router)
  1000000n,                        // 1 USDC (6 decimals)
  3600                             // Valid for 1 hour
);

// 2. Any relayer/Agent can submit on-chain (pays gas)
const txHash = await agent.payWithPermit(
  PAYNODE_ROUTER_ADDRESS,
  PAYER_WALLET_ADDRESS,            // Token holder
  USDC_ADDRESS,
  MERCHANT_ADDRESS,
  1000000n,                        // Amount
  orderId,                         // bytes32
  deadline, v, r, s
);
```

## 🏪 For Merchants (Express Middleware)

The `x402_gate` middleware protects your Express routes by enforcing USDC payment verified on-chain.

```typescript
import express from 'express';
import { x402_gate } from '@paynodelabs/sdk-js';

const app = express();

app.get('/api/ai-inference', x402_gate({
  rpcUrls: ['https://mainnet.base.org'],
  chainId: 8453,
  contractAddress: '0xB587Bc36aaCf65962eCd6Ba59e2DA76f2f575408',
  merchantAddress: '0xYourWallet...',
  tokenAddress: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913',  // USDC
  currency: 'USDC',
  price: '2.50',
  decimals: 6,
}), (req, res) => {
  // This code ONLY runs after verified on-chain payment
  res.json({
    result: 'The meaning of life is 42.',
    receipt: req.paynode.receiptHash,
    orderId: req.paynode.orderId,
  });
});

app.listen(3000);
```

### Middleware Headers

When a request lacks the `x-paynode-receipt` header, the middleware returns `402` with these response headers:

| Header | Description |
| :--- | :--- |
| `x-paynode-contract` | Router contract address |
| `x-paynode-merchant` | Merchant wallet address |
| `x-paynode-amount` | Required amount (smallest unit) |
| `x-paynode-currency` | Token symbol (e.g. `USDC`) |
| `x-paynode-token-address` | ERC20 token address |
| `x-paynode-chain-id` | Chain ID (e.g. `8453`) |
| `x-paynode-order-id` | Generated order ID |

## 🛡️ Token Whitelist

The Verifier includes a built-in token whitelist to prevent fake-token attacks. By default, it accepts:

| Chain | Tokens |
| :--- | :--- |
| Base Mainnet (8453) | USDC `0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913` |
| Base Sepolia (84532) | Mock USDC `0xeAC1f2C7099CdaFfB91Aa3b8Ffd653Ef16935798` |

### Customization

```typescript
import { PayNodeVerifier, ACCEPTED_TOKENS } from '@paynodelabs/sdk-js';

// Use chain default (auto-detected from chainId)
const verifier = new PayNodeVerifier({
  rpcUrls: ['https://mainnet.base.org'],
  chainId: 8453,
  // acceptedTokens is auto-loaded from ACCEPTED_TOKENS[8453]
});

// Custom whitelist (e.g. only accept DAI)
const verifier = new PayNodeVerifier({
  rpcUrls: ['https://mainnet.base.org'],
  chainId: 8453,
  acceptedTokens: ['0x50c5725949A6F0c72E6C4a641F24049A917DB0Cb']  // DAI on Base
});

// Explicitly disable whitelist (testing only)
const verifier = new PayNodeVerifier({
  rpcUrls: ['http://localhost:8545'],
  acceptedTokens: []  // No whitelist check
});
```

## 🔒 Production: Redis Idempotency

For production multi-instance deployments, use `RedisIdempotencyStore` instead of the default in-memory store:

```typescript
import Redis from 'ioredis';
import { x402_gate, RedisIdempotencyStore } from '@paynodelabs/sdk-js';

const redis = new Redis(process.env.REDIS_URL);

app.get('/api/data', x402_gate({
  // ...other options
  store: new RedisIdempotencyStore(redis, 'paynode:tx:'),
}), handler);
```
